# 售后服务记录系统 - 使用指南

> 员工版 | 管理员版

---

## 员工：每次服务后怎么做

### 步骤 1：填写记录表

完成服务后，填写《售后服务记录表》（`FAQ/07_售后服务记录表.md`）

### 步骤 2：发到工作群

将填写好的记录**发到工作群**，并 **@管理员**

### 步骤 3：完成

就这么简单！管理员会处理后续。

---

## 管理员：收到记录后怎么做

### 步骤 1：收集记录

从工作群收集员工发来的售后服务记录

### 步骤 2：复制到待分析目录

将每条记录保存为单独文件：
```
FAQ/待分析/YYYY-MM-DD_[客户名]_[问题类型].md
```

例如：
```
FAQ/待分析/2026-04-10_华强化工厂_黑点问题.md
FAQ/待分析/2026-04-11_金峰塑胶_温度调整.md
```

### 步骤 3：通知 AI 分析

在 Claude Code 中运行：
```
/analyze-records
```

### 步骤 4：审核 AI 输出

AI 分析后，更新建议会保存到 `FAQ/待审核/` 目录

管理员审核后，手动合并到主 FAQ 目录

### 步骤 5：完成归档

已处理的记录移到 `FAQ/已归档/`

## FAQ 知识库自动更新流程

```
员工填写 → 发到群 → @管理员
    ↓
管理员收集 → 保存到待分析目录
    ↓
每月运行 /analyze-records
    ↓
AI 分析 → 输出更新建议到待审核目录
    ↓
管理员审核 → 合并到主 FAQ
    ↓
员工培训 / 客户自助查询
```

## 常见问题

**Q：记录表太复杂，可以简化吗？**

A：可以。但技术要点（根因、参数调整、经验）必须填写，这是 AI 学习的关键。

**Q：忘记填写怎么办？**

A：尽快补填，越早越好。最好当日完成。

**Q：客户问题知识库没有怎么办？**

A：正常填写，AI 会分析并决定是否需要新增 FAQ 条目。

**Q：AI 分析出错怎么办？**

A：管理员审核环节会修正，放心使用。
