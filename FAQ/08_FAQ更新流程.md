# FAQ 自动更新流程

> 售后服务记录 → AI 分析 → 知识库更新

---

## 闭环流程图

```
员工完成服务
    ↓ 填写记录表，发到工作群
工作群通知 [@管理员]
    ↓
管理员汇总
    ↓ 每月/每批
AI 分析记录
    ↓ 提取新问题、新方案、新参数
更新 FAQ 知识库
    ↓
新 FAQ 上线
    ↓
员工培训 / 客户自助查询
```

---

## 角色分工

| 角色 | 职责 |
|------|------|
| 服务人员 | 完成服务后填写《售后服务记录表》，发到工作群 |
| 管理员 | 收集记录表，汇总到 `FAQ/待分析/` 目录 |
| AI (Claude Code) | 每月分析汇总，自动更新 FAQ 知识库 |

---

## 文件流转

```
员工填写 → 工作群 @管理员
    ↓
管理员复制到：
FAQ/待分析/YYYY-MM-DD_[客户名]_[问题类型].md
    ↓
每月末：AI 批量分析
    ↓
AI 输出：
FAQ/新增内容/新FAQ条目.md
FAQ/参数更新/新工艺参数.md
FAQ/故障案例/新故障案例.md
    ↓
管理员审核后 → 合并到主 FAQ 目录
```

---

## AI 分析任务清单

当管理员运行 `/analyze-records` 时，AI 会：

- [ ] 读取 `FAQ/待分析/` 目录下所有新记录
- [ ] 提取每条记录的问题分类和技术要点
- [ ] 对比现有 FAQ，识别新增或需要更新的内容
- [ ] 生成更新建议，输出到 `FAQ/待审核/`
- [ ] 特别关注：
  - 新故障类型 → 更新 `05_故障处理.md`
  - 新参数调整 → 更新 `03_工艺参数.md`
  - 新解决方案 → 更新对应 FAQ 条目
  - 高频问题 → 标记优先更新

---

## 待分析目录

创建 `FAQ/待分析/` 目录用于收集待处理记录：

```
FAQ/
├── 待分析/              ← 员工提交的原始记录
├── 待审核/              ← AI 分析后的更新建议
└── 已归档/              ← 已合并到主 FAQ 的记录
```

---

## 启动方式

管理员在 Claude Code 中运行：

```
/analyze-records
```

AI 会自动读取 `FAQ/待分析/` 目录，执行分析并输出更新建议。

---

*此流程确保知识库持续迭代，不漏掉任何实际案例！*
