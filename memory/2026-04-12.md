## 🔧 Claude-to-IM CardKit 9499 错误修复（2026-04-12 12:50）

### 问题现象
飞书回复提示"请升级至最新版本客户端"，卡片更新失败（code 9499: Invalid parameter value）

### 根因分析
飞书 CardKit v1 API vs v2 的差异导致的问题：

| 问题 | 原因 |
|------|------|
| schema 2.0 被拒绝 | v1 API 只支持 schema 1.0 |
| streamingMode.set 警告 | v1 SDK 没有这个 API |
| text_size 'notation' 无效 | v1 不支持该值 |
| tag: 'markdown' 失败 | v1 不支持 markdown 元素，只支持 text |
| card 双层序列化 | SDK 会序列化整个 data 对象，传字符串会导致双层 JSON |

### 修复内容（Claude-to-IM/src/lib/bridge/）

**feishu-adapter.ts:**
- `cardkit.v2` → `v1`（所有 API）
- 删除不存在的 `streamingMode.set` 调用
- `card.update` 的 data: `{ card: string }` → `{ card: JSON.parse(string) }`（传对象）

**feishu.ts (buildFinalCardJson / buildCardContent / buildPermissionButtonCard):**
- `schema: '2.0'` → `'1.0'`
- `tag: 'markdown'` → `'text'`
- `text_size: 'notation'` → `'small'`

### 重建命令
```bash
cd C:\Users\pc\.claude\Claude-to-IM && npm run build
cd C:\Users\pc\.claude\skills\claude-to-im && npm run build
```

### 当前状态（PID 6244）
- 运行中，CardKit update API 修复成功（9499 消除）
- **新问题**：`im.message.create` 发送 interactive 卡片报 `230099 parse card json err`
- 最简格式 `{"schema":"1.0","body":{"elements":[{"tag":"text","content":"..."}]}}` 仍然失败

### 后续排查方向
- `im.v1.messages.create` 的 `content` 字段可能需要传 JSON 对象而非字符串
- 或飞书卡片需要 `{ type: 'template', data: {...} }` 结构而非直接卡片 JSON
- 建议：用 curl 直接测试 API 格式，排除 SDK 序列化问题

### 飞书 CardKit v1 vs v2 关键差异
- v1 schema: `1.0`，v2: `2.0`
- v1 元素标签：`text`/`hr`/`image`/`button`/`column_set`，不支持 `markdown`
- v1 text_size：`normal`/`small`/`caption`，不支持 `notation`
- v1 update API：`PUT /cardkit/v1/cards/:card_id`，data 格式为 `{ card: object }`
