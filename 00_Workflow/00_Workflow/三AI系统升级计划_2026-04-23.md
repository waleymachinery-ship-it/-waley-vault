## 二、P0 — MCP协议集成（Model Context Protocol）

### 2.1 项目背景

MCP是Anthropic推出的开放协议，用于标准化LLM与外部工具/数据源的连接。

**对我们三AI体系的意义：**
- Jarvis/Hermes/Claude 之间的通信可以用MCP标准化
- 可以通过MCP Server暴露工具给其他AI调用
- 已有成熟Python/JS SDK，社区生态丰富

### 2.4 快速开始示例

**最简单的MCP Server（filesystem示例）：**

```python
# mcp_filesystem_server.py
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("伟力Vault文件服务")

@mcp.tool()
def read_vault_file(path: str) -> str:
    """读取Vault知识库文件"""
    with open(path, 'r', encoding='utf-8') as f:
        return f.read()

@mcp.resource("vault://memory")
def vault_memory() -> str:
    """返回Vault记忆同步状态"""
    return "Z_Memory_Sync v220 | 三AI在线"
```

**运行服务器：**
```bash
python mcp_filesystem_server.py
```

## 三、P1 — 多Agent协作模式学习

### 3.1 CrewAI（重点参考）

**学习价值：** CrewAI的"角色+任务+自主协作"模式可直接映射到我们的Jarvis/Hermes/Claude分工。

| 资源 | 链接 |
|------|------|
| 官网 | https://docs.crewai.com |
| GitHub | https://github.com/crewAIInc/crewAI |
| PyPI | https://pypi.org/project/crewai/ |
| 安装 | `pip install crewai` |

**crewAI基础概念：**
```
Agent（角色）= Jarvis客服 / Hermes调度 / Claude执行
Task（任务）  = 故障报告处理 / BOM验证 / 文档生成
Crew（团队）  = 三AI协作团队
```

**示例代码：**
```python
from crewai import Agent, Crew, Task

# 定义客服角色
support_agent = Agent(
    role="伟力机械客服",
    goal="准确回答客户问题，引导留资",
    backstory="你是伟力机械的24/7 AI客服"
)

# 定义任务
support_task = Task(
    description="回答客户关于产品的问题",
    agent=support_agent
)

# 组建团队
crew = Crew(agents=[support_agent], tasks=[support_task])
result = crew.kickoff()
```

### 3.2 LangGraph（状态机参考）

**学习价值：** LangGraph的状态机流设计适合建模故障报告处理流程。

| 资源 | 链接 |
|------|------|
| PyPI | https://pypi.org/project/langgraph/ |
| 文档 | https://docs.langchain.com/oss/python/langgraph |
| GitHub | 126K+ stars |
| 安装 | `pip install -U langgraph` |

**LangGraph状态机示例（故障报告流程）：**
```python
from langgraph.graph import StateGraph, START, END

def fault_report_graph():
    workflow = StateGraph(dict)
    
    # 定义状态节点
    workflow.add_node("receive", receive_report)
    workflow.add_node("analyze", analyze_fault)
    workflow.add_node("respond", respond_to_customer)
    workflow.add_node("archive", archive_record)
    
    # 定义流程
    workflow.add_edge(START, "receive")
    workflow.add_edge("receive", "analyze")
    workflow.add_edge("analyze", "respond")
    workflow.add_edge("respond", "archive")
    workflow.add_edge("archive", END)
    
    return workflow.compile()
```

**背景：** 当前BOM验证已达97.67%，但小组件识别率低，engineering-drawing-extractor可能提供补充思路。

| 资源 | 链接 |
|------|------|
| GitHub | https://github.com/Bakkopi/engineering-drawing-extractor |

**功能：**
- 自动分离工程图纸与表格
- OCR提取图号、作者、标题等信息
- 去除不必要线条、边框、注释

**安装：**
```bash
git clone https://github.com/Bakkopi/engineering-drawing-extractor
cd engineering-drawing-extractor
pip install -r requirements.txt
```

**适用场景：**
- 处理3DX技术资料中的图片（image4/5/6.jpeg）
- 提取图纸中的尺寸标注文字
