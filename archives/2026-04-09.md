## 问题一：Hermes 身份认知错误

### 错误：以为 Hermes 是 pip 包

**现象：** 最初认为 Hermes 可以通过 `pip install hermes-ai` 直接安装。

**实际操作：**
```bash
# 在 WSL 里执行
curl -s https://bootstrap.pypa.io/get-pip.py | python3 - --index-url https://pypi.tuna.tsinghua.edu.cn/simple/ --break-system-packages
pip3 install hermes-ai -i https://pypi.tuna.tsinghua.edu.cn/simple/ --break-system-packages
```

**结果：** 安装成功，但安装的是 **PyPI 上的 metadata harvesting 工具**（hermes-ai 0.9.1），而非 NousResearch 的 AI Agent 框架。两者是完全不同的软件。

**发现过程：**
- `hermes --version` 输出 `0.9.1`，不是 AI Agent 版本号
- `hermes --help` 显示只有 `help, version, init, clean, harvest, process, curate, deposit, postprocess` 子命令
- `hermes doctor` 报错 `invalid choice`
- 对比 SKILL.md 中描述的 Hermes Agent（支持 `chat, model, tools, skills, gateway` 等命令），确认为不同软件

**教训：** Hermes Agent (NousResearch) 不是 pip 包，需要通过源码或官方安装脚本安装。

## 问题二：WSL Ubuntu 文件系统异常

### 症状

```bash
# 执行以下命令，WSL 均无响应或报错
wsl -e bash -c "echo $HOME"      # 输出为空
wsl -e bash -c "echo $PATH"      # 输出为空
wsl -e bash -c "ls /home/shane/" # 显示 Windows 目录 C:\Users\pc\.hermes\ 的内容
```

**具体表现：**
- `ls /home/shane/` 返回的是 Windows 目录 `C:\Users\pc\.hermes\` 的内容（AGENTS.md、SOUL.md、MEMORY.md 等）
- HOME 和 PATH 环境变量为空
- 任何 `wsl -e bash -c "..."` 均报 `WSL (N - Relay) ERROR: CreateProcessCommon:800: execvpe(bash) failed: No such file or directory`

### 根因

WSL Ubuntu 的根文件系统（rootfs）没有正确挂载，Linux 子系统看到的 `/home/shane/` 实际指向了 Windows 目录。这是非正常状态。

### 尝试的修复方法

| 方法 | 命令 | 结果 |
|------|------|------|
| 注销 Ubuntu | `wsl --unregister Ubuntu` | 命令卡住，最终 SIGKILL |
| 重启 WSL | `wsl --shutdown` | 命令卡住，最终 SIGKILL |
| 重装 Ubuntu | `wsl --install Ubuntu` | 安装过程卡住，最终 SIGKILL |
| WSL 更新 | `wsl --update` | 命令执行后无响应 |
| WSL 列表 | `wsl --list --verbose` | 命令卡住超时 |

### 结论

WSL 子系统整体挂起/损坏，Windows 层面的 WSL2 虚拟机管理器异常，所有 wsl.exe 管理命令均无法正常执行。
